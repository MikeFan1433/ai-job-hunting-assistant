# 🚀 本地运行指南

## ✅ 快速启动

### 方法 1: 使用启动脚本（推荐）

```bash
cd "AI Job Hunting Assistant"
./start_local_services.sh
```

### 方法 2: 手动启动

**终端 1 - 启动后端:**
```bash
cd "AI Job Hunting Assistant"
python3 -m uvicorn workflow_api:app --host 0.0.0.0 --port 8000 --reload
```

**终端 2 - 启动前端:**
```bash
cd "AI Job Hunting Assistant/frontend"
npm run dev
```

## 📱 访问链接

### 前端应用
- **主要链接**: http://localhost:3000
- **备用端口**: http://localhost:3001 或 http://localhost:5173

### 后端 API
- **API 地址**: http://localhost:8000
- **健康检查**: http://localhost:8000/api/v1/health

## 🔧 确保服务长期运行

### 使用 nohup（后台运行）

**启动后端（后台）:**
```bash
cd "AI Job Hunting Assistant"
nohup python3 -m uvicorn workflow_api:app --host 0.0.0.0 --port 8000 --reload > /tmp/backend.log 2>&1 &
```

**启动前端（后台）:**
```bash
cd "AI Job Hunting Assistant/frontend"
nohup npm run dev > /tmp/frontend.log 2>&1 &
```

### 检查服务状态

```bash
# 检查后端
lsof -ti:8000 && echo "后端运行中" || echo "后端未运行"

# 检查前端
lsof -ti:3000,3001,5173 && echo "前端运行中" || echo "前端未运行"
```

### 查看日志

```bash
# 后端日志
tail -f /tmp/backend.log

# 前端日志
tail -f /tmp/frontend.log
```

## 🛑 停止服务

```bash
# 停止后端
lsof -ti:8000 | xargs kill

# 停止前端
lsof -ti:3000,3001,5173 | xargs kill

# 停止所有服务
lsof -ti:8000,3000,3001,5173 | xargs kill
```

## 📋 服务信息

### 后端服务
- **端口**: 8000
- **地址**: http://localhost:8000
- **API 文档**: http://localhost:8000/docs
- **健康检查**: http://localhost:8000/api/v1/health

### 前端服务
- **端口**: 3000 (或 3001/5173，如果 3000 被占用)
- **地址**: http://localhost:3000
- **开发模式**: 支持热重载

## 💡 提示

1. **端口被占用**: 如果端口被占用，服务会自动使用下一个可用端口
2. **后台运行**: 使用 `nohup` 和 `&` 可以让服务在后台运行
3. **查看日志**: 日志文件保存在 `/tmp/backend.log` 和 `/tmp/frontend.log`
4. **重启服务**: 如果服务停止，运行 `./start_local_services.sh` 重新启动
