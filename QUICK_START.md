## 🔧 如果页面打不开

### 1. 检查服务器是否运行
```bash
lsof -i:8000
```

如果没有任何输出，说明服务器没有运行。

### 2. 手动启动服务器
```bash
cd "/Users/mikefan/Desktop/AI Architect - Superlinear/AI Job Hunting Assistant"
python3 -m uvicorn workflow_api:app --host 0.0.0.0 --port 8000 --reload
```

### 3. 检查端口是否被占用
```bash
lsof -i:8000
```

如果端口被占用，可以：
- 杀死占用进程：`kill -9 $(lsof -ti:8000)`
- 或使用其他端口：`--port 8001`

### 4. 检查错误信息
查看运行服务器的终端窗口，查看是否有错误信息。

## 📝 常见问题

### 问题 1: 连接被拒绝
**原因：** 服务器没有运行
**解决：** 启动服务器（见上方）

### 问题 2: 404 错误
**原因：** 静态文件路径问题
**解决：** 确保 `static/` 目录存在且包含 `index.html`

### 问题 3: 500 错误
**原因：** 后端代码错误
**解决：** 查看终端错误信息，检查导入的模块是否都安装

### 问题 4: CORS 错误
**原因：** 跨域问题
**解决：** 检查 `workflow_api.py` 中的 CORS 配置
